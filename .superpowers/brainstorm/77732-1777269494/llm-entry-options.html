A. 当前弹窗原生完成

最贴近你的诉求，`rainbond-ui` 自己承接四个入口和前三个表单。

方案 B
创建组件
扩展与中间件 → 大模型
四个入口保留
点击前三个后，不写原生表单，而是在当前 Modal 内嵌 AI Engine 插件页面。
插件内容直接嵌进来
部署模型 / 模型市场 / 启动实例 全走插件现成逻辑

B. 当前弹窗里嵌插件

复用插件最多，但交互感受会更像“在弹窗里再打开一个插件页面”。

方案 C
创建组件
扩展与中间件 → 大模型
四个入口

魔搭 / 上传 / 本地 / 模型市场

点击任何一个都直接跳插件页，用 query 控制落点

改动最少，但前三个不会在当前弹窗里完成。

C. 四个入口都做跳转

实现成本最低，但不符合你刚刚确认的“前三个留在当前弹窗里完成”。

我推荐 A

  • 最符合你已经确认的交互目标。
  • `本地` 还能继续沿用现有 `modelKey` 自动打开部署抽屉的能力。
  • 用户从“创建组件”进入时，不会被突然打断到插件页再继续填表。

A 的代价

  • `rainbond-ui` 里要补几条 AI Engine 调用接口，而不是只保留读取已下载模型。
  • 文案、状态和表单会有一小部分和插件逻辑同步维护。
